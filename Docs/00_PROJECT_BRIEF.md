# docs/00_PROJECT_BRIEF.md — Vision du portfolio

## Nom du projet

Portfolio web personnel moderne.

Nom final à confirmer.

## Pitch

Créer un portfolio web ultra moderne permettant de présenter un profil tech/créatif, des projets personnels ambitieux, des projets d’étude, un CV et des hobbies, avec un rendu premium adapté à GitHub Pages.

Le site doit montrer à la fois :

* des compétences techniques ;
* une capacité à concevoir des projets complets ;
* une sensibilité créative ;
* une progression professionnelle ;
* une identité personnelle forte.

## Public cible

Le portfolio doit pouvoir être consulté par :

* recruteurs ;
* entreprises tech ;
* formateurs ;
* collègues développeurs ;
* visiteurs curieux ;
* contacts venant de LinkedIn ou GitHub.

## Ton général

Le ton doit être professionnel, moderne, clair et humain.

Il ne doit pas être trop corporate ni trop froid.

Le site doit donner une impression de sérieux, mais aussi de créativité.

## Identité recherchée

Mots-clés :

* moderne ;
* premium ;
* fluide ;
* tech ;
* lumineux ;
* créatif ;
* dynamique ;
* sobre ;
* élégant ;
* immersif.

## Contenus principaux

### Page d’accueil

La page d’accueil doit être la vitrine principale.

Elle contient :

* un grand hero ;
* un carrousel avec trois visuels majeurs ;
* une section résumé du profil ;
* une section projets mis en avant ;
* une section compétences ou parcours ;
* un call-to-action vers le CV ou les projets ;
* un footer avec réseaux sociaux.

### Carrousel d’accueil

Le carrousel doit contenir trois slides :

1. Parcours professionnel
   Mettre en avant l’expérience, le profil technique et la progression.

2. Projets personnels
   Mettre en avant BrawlCards: Origins et Les Terres de Verdelya.

3. Créativité / hobbies / piano
   Mettre en avant l’aspect créatif, musical et personnel.

Chaque slide doit contenir :

* un titre fort ;
* une phrase courte ;
* un bouton d’action ;
* un visuel ou une illustration ;
* un effet moderne mais pas trop lourd.

### Pages projets personnels

Créer une page dédiée à chaque projet :

* BrawlCards: Origins ;
* Les Terres de Verdelya.

Chaque page projet doit contenir :

* titre ;
* pitch ;
* contexte ;
* fonctionnalités principales ;
* stack technique ;
* captures ou placeholders ;
* statut du projet ;
* liens éventuels ;
* apprentissages ;
* prochaines évolutions.

### Page projets d’étude

La page projets d’étude regroupe :

* SoundTrackify ;
* ZombieLand.

Chaque projet doit avoir une section ou une carte détaillée.

### Page CV

La page CV doit être claire, lisible et professionnelle.

Elle doit contenir :

* titre ;
* résumé ;
* expérience ;
* compétences ;
* projets significatifs ;
* formation ;
* lien de téléchargement PDF si disponible ;
* CTA vers LinkedIn ou contact.

Informations connues à intégrer :

* 15 ans d’expérience en entreprise ;
* 13 ans comme Webmaster ;
* 2 ans comme Technical Leader.

### Page hobbies

La page hobbies doit présenter principalement le piano.

Elle doit contenir :

* une introduction personnelle ;
* une section piano ;
* quelques vidéos YouTube intégrées ou affichées sous forme de cartes ;
* éventuellement d’autres hobbies plus tard.

Les vidéos YouTube doivent être configurables depuis un fichier de données.

### Footer

Le footer doit contenir les réseaux :

* LinkedIn ;
* GitHub ;
* Discord ;
* Patreon ;
* YouTube ;
* autres si fournis plus tard.

Les liens non fournis doivent être remplacés par des placeholders `TODO`.

## Informations manquantes à demander plus tard

Ne pas bloquer le développement si ces informations manquent. Utiliser des placeholders propres.

À compléter plus tard :

* nom exact affiché sur le site ;
* photo/avatar ;
* liens sociaux ;
* URL des vidéos YouTube ;
* captures d’écran des projets ;
* lien CV PDF ;
* descriptions détaillées des Terres de Verdelya et ZombieLand ;
* palette finale si une préférence apparaît ;
* typographie finale si une préférence apparaît.
